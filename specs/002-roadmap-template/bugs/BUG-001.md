# Bug Report: Backend API Data Missing Rich Metadata for v2 Roadmap Template

**Type**: Spec gap / Implementation drift
**Severity**: High
**Feature**: 002-roadmap-template
**Reported**: 2026-06-14

## Description
When integrating the new v2 roadmap template into the actual web application frontend (`web/src/pages/courses/CourseDetail.tsx`), the dynamic data fetched from the backend API (`/api/courses/{course_id}`) only contains a flat list of nodes with basic properties (`node_id`, `title`, `position`, `status`, `estimated_minutes`). 

However, the v2 roadmap template requires a rich hierarchical structure (Phases -> Topics), resource arrays, dependencies (`topic_graph`), and variants. Because this data is missing from the database/API payload, the frontend's mapping function is starved of data, resulting in a fallback linear view under a single "General Foundations" phase. This breaks the intended UI (the astronaut mascot disappears, missing descriptions, static "variants", etc.).

## Artifact Traceability

### spec.md
- **Affected user story**: 
  - [US1] Roadmap UI Core [P1]
  - [US3] Topic Details Panel [P2]
  - [US5] Variant Filtering [P3]
- **Gap identified**: The spec assumed the frontend would consume a perfect JSON matching `roadmap_template_v2.json`. It completely missed the backend data integration and data modeling requirements needed to update the database schema and API to actually serve this v2 data format.

### plan.md
- **Affected sections**: "Storage", "Input", "Technical Context"
- **Impact**: The plan needs to be updated to include backend migration. The backend API (`courses.py`) and repository layers (`course_repo.py`) need to be modified to store and serve the new schema.

### tasks.md
- **False completions**: None directly, as the frontend tasks were completed against mock data as planned.
- **Missing tasks**: 
  - Update MongoDB data seeding (`backend/scripts/initial_sync.py`) to support hierarchical phases, tags, and variants.
  - Update `/api/courses/{course_id}` endpoint in `backend/app/api/courses.py` to return the full rich data structure.
  - Update database models/validation to match the v2 schema.

## Root Cause Analysis
The frontend UI was built in a silo against mock static JSON files (e.g., `src/data/course-roadmap.json`), but the existing production database and API were never upgraded to provide this complex data. When the frontend was wired to the real API, the data mapping layer could only extract basic titles and IDs, dropping all the rich features of the v2 design.
